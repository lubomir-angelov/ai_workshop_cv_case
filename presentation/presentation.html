The Problem

Business context

Self-checkout stores need to automatically track when customers pick up or put down items at shelves — without manual review of hours of CCTV footage.

The core challenge
  • Video is long and mostly uneventful — events are rare
  • Pickup and putdown are near-identical visually — only temporal direction separates them
  • Multiple people, occlusions, cluttered shelves
  • 4K footage (3840×2160) at 20 FPS
  • Privacy constraints — no face identification
Technical framing

This is Temporal Action Detection on untrimmed video — not simple action recognition.

Output per video:
  clip_id
  event_type: pickup | putdown
  t_start  (seconds)
  t_end    (seconds)
  score
Key insight

Pickup and putdown are time-reverses of each other. Any method that discards temporal order will fail to distinguish them.

pickup:  shelf/surface → hand
putdown: hand → shelf/surface

Layer 0A — Person Triage & Active Spans

Goal

Answer three questions without detecting any events:

  • Is this clip technically usable?
  • Does it contain a person?
  • When is any person visible? (active spans)

Algorithms

  • YOLO11n — lightweight person detector
  • ByteTrack — multi-object tracking
  • Sampled at 1 FPS (stride 20 on 20 FPS source)
  • vid_stride = max(1, round(source_fps / target_fps))

Stable-track rule

PERSON_PRESENT when:
  duration  ≥ 0.75 s (source timestamps)
  confident observations ≥ 2
  not isolated accidental detection

Layer 0B — Pose Tracking & Interaction Proposals

Goal

For each active span, find when a tracked person's wrist is likely interacting with a shelf. Produces high-recall candidates — not event predictions.

Algorithms

  • YOLO Pose — 17-keypoint body pose estimation
  • Wrist keypoints tracked per actor at ~8 FPS (stride 2)
  • Actual effective sample rate: 8.04 FPS
  • Actor association via IoU between person bbox and pose bbox

Fixed shelf regions (store_camera_01)

3 shelf/surface polygons configured and version-controlled. Expansion margin: 60 px. Clamped to 3840×2160 frame bounds.

Task 8 — Shared Two-Pass Temporal Evaluator

Pass 1 — Class-aware matching

For each clip and event type, match predictions to ground truth of the same type using Hungarian assignment.

Match criterion (either):
  tIoU ≥ 0.3  or  ≥ 0.5
  midpoint within ±1 second

→ Precision / Recall / F1 per class

Pass 2 — Class-agnostic confusion

Match ignoring type, then compare types after matching:

GT pickup  + pred pickup  → correct
GT pickup  + pred putdown → CONFUSION
GT putdown + pred pickup  → CONFUSION
GT putdown + pred putdown → correct

Required metrics

  • Precision / Recall / F1 at tIoU 0.3 and 0.5
  • F1 at midpoint tolerance ±1 s
  • pickup→putdown confusion rate
  • putdown→pickup confusion rate
  • Start / end time MAE
  • False positives per video-hour
  • Stage B proposal recall
  • Multi-item event recall
  • mAP at selected tIoU thresholds (optional)

Evaluation slices

  • All official event rows
  • High/med confidence only
  • Low confidence only
  • Normal vs hard_case=true
  • Single-person vs multi-person

One evaluator — all systems

Track A, Track B1, Track B2, Layer 2 (Qwen standalone), and Layer 3 (fusion) all run through the same evaluator, enabling a fair head-to-head comparison.

Precise Event Definitions

Pickup

A person removes an item from a shelf or surface and takes it into their hand(s), so that the item leaves its resting place and is carried or held.

Transition: shelf/surface → hand
t_start: onset of final purposeful action
t_end:   item stably held/carried

Putdown

A person places an item they were holding onto a shelf or surface and releases it so that it rests there again.

Transition: hand → shelf/surface
t_start: onset of final purposeful action
t_end:   item released & stably resting

Hard negatives (NOT events)

Touching without removing Reaching past / browsing Standing near shelves Restocking (new goods) Occluded actions Empty / no-person clips

Edge-case rules

  • Two items simultaneously → two event rows
  • Pickup then immediate return → two ordered events
  • Occluded / out-of-frame → ignore interval (not labeled)
  • Multiple actors → all labeled
  • Ambiguous but visible → keep with confidence=low

Layer 1 Track A — Deterministic State Machine

Philosophy

No learned weights for core detection. Combines geometric pose evidence with lightweight appearance classifiers and a repeating finite state machine.

State machine

OUTSIDE
  → APPROACHING
  → CONTACT
  → TRANSFER / STABILIZATION
  → WITHDRAWING
  → OUTSIDE  (→ can repeat)

One candidate can cycle multiple times → multiple events.

Decision rules

Pickup: hand approaches + transfer visible + hand becomes carrying + shelf indicates removal

Putdown: hand approaches while carrying + hand becomes empty + shelf indicates placement

Appearance features (per transition)

Three frames sampled: pre-contact · contact/transfer · post-stabilization

  • hand_before / hand_after crops
  • shelf_before / shelf_after local patches
  • Frozen embeddings: DINOv2-small / SigLIP / CLIP

Lightweight classifiers

Hand state
empty | carrying | uncertain
Shelf transition
removed | placed | unchanged | uncertain

Models: logistic regression, small MLP, or gradient-boosted trees.

Boundary method tracking

If precise transfer boundaries can't be estimated, fallback is recorded:

boundary_method = WRIST_REGION_FALLBACK

Layer 1 Track B1 — VideoMAE Window Classifier

Model: VideoMAE-Small

Masked video autoencoder pretrained on Kinetics. Classification head fine-tuned on actor-conditioned windows from our dataset.

Actor-conditioned crop

crop = union(actor bboxes across window)
       + active shelf/surface region
       + 15–20% margin

→ simultaneous people processed
  independently (no cross-person leakage)

Window parameters

window duration:  2.5 s
stride:           0.5 s
sampled frames:   16 (chronological)
classes:          background | pickup | putdown

Training gates

  1. Gate A: visual inspection — render 20+ samples with frames, crop box, label, timestamps
  2. Gate B: tiny overfit — overfit 8–16 samples before full training
  3. Gate C: baseline — frozen backbone + trained head; checkpoint by validation event F1

Inference & decoding

  1. Slide window with 0.5 s stride
  2. Smooth class probabilities chronologically
  3. Detect distinct per-class peaks
  4. Merge only same-type duplicates
  5. Never merge adjacent pickup ↔ putdown
  6. Export canonical event intervals

Critical constraint

Frames must always be in chronological order. An unordered bag of frames cannot distinguish pickup from putdown.

Layer 1 Track B2 — Cached Features + TCN

Two-stage design

Stage 1 (GPU — run once)
VideoMAE-Small encodes overlapping 2 s micro-clips → cached embeddings [T, D]
↓ cached as .npz per actor/candidate
Stage 2 (CPU — fast iteration)
Small TCN trained on cached features → per-timestep class scores

Why cache?

Feature extraction is expensive GPU work. Once cached, many temporal-head experiments run on CPU in minutes — enabling rapid iteration without re-running VideoMAE.

TCN architecture

VideoMAE sequence [T × D]
        ↓
Linear projection → 256d
        ↓
Conv1D residual block (k=3)
        ↓
Dilated Conv1D block (d=2)
        ↓
Dilated Conv1D block (d=4)
        ↓
Per-timestep: background | pickup | putdown

Temporal label assignment

  • Center inside pickup interval → pickup
  • Center inside putdown interval → putdown
  • Inside ignore interval → ignore (loss = 0)
  • Otherwise → background

Loss & confidence weights

Focal loss / weighted cross-entropy
high/med confidence:  weight = 1.0
low confidence:       weight = 0.5
ignore interval:      weight = 0.0

Layer 2 — Standalone Qwen3.6-27B VLM Detector

Independence constraint

Layer 2 must not receive Layer 1 predictions. It scans active-span windows independently, enabling a fair comparison.

Input windows

duration:    8.0 s
stride:      4.0 s (50% overlap)
sampling:    4 FPS → 32 frames/window

Each window:
  - chronological video frames
  - frame numbers + relative timestamps
  - clip_id + window start time

Structured JSON response

{
  "events": [{
    "type": "pickup",
    "t_start": 2.4,
    "t_end": 3.7,
    "item_count": 1,
    "visible": true,
    "confidence": 0.82
  }]
}

Post-processing

  • Pydantic schema validation (retry once on failure)
  • Convert relative window times → source clip times
  • Merge only same-type overlapping predictions from adjacent windows
  • Expand item_count=2 into two canonical rows
  • Raw responses preserved in raw_responses.jsonl

Prompt must include

  • Full operational event definitions
  • Edge-case rules (two items, immediate pickup/putdown)
  • Negative examples (touch-only, restocking)
  • Temporal-order instruction
  • Strict JSON schema

Hardware logged per run

Model ID, quantization, GPU, peak VRAM, inference speed, and parse failure rate are all recorded for reproducibility.

Layer 3 — Qwen Verification & Deterministic Fusion

Purpose

For each Layer 1 event prediction, send a short context clip to Qwen and ask it to verify the event. Layer 1 provides timing; Qwen provides semantic confirmation.

Verification window

4 s before Layer 1 t_start
        through
2 s after Layer 1 t_end

Pre-context distinguishes genuine putdown (item was held) from restocking (item is new).

Verification response

{
  "event_visible": true,
  "event_present": true,
  "event_type": "pickup",
  "item_count": 1,
  "confidence": 0.91,
  "reason_code": "ITEM_LEAVES_SURFACE_WITH_HAND"
}

Deterministic fusion rules
